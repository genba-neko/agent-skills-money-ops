# #27 pdf_to_json 特定口座年間取引報告書 スキーマ精度改善 [完了 PR#28 2026-04-23]

## 対象 issue

[#27](https://github.com/genba-neko/agent-skills-money-ops/issues/27)

---

## 背景・問題

`src/money_ops/converter/pdf_to_json.py` の JSON スキーマ（`_EXTRACTION_PROMPT`）が不完全で、
配当等セクションの フィールド誤認識が発生していた。

### 具体的バグ

ウィブル証券の検証で発覚:

- ⑧ 外国株式等の配当等（58,768円）と ⑭ 国外公社債等の配当等（2,781円）が
  `外国株式等の配当等` の 1 フィールドに合算されて 61,549 円と抽出されていた
- スキーマに「上記以外のもの」セクション（⑩〜⑮）が存在せず、
  LLM がフィールドをまとめて誤解釈

---

## 修正内容

### 1. スキーマ構造の再設計

配当等を正しく 2 セクションに分割:

```
配当等
├── 特定上場株式等（④〜⑨）
│   ├── 上場株式の配当等 ④
│   ├── 特定株式投資信託の収益の分配等 ⑤
│   ├── 投資信託等の収益の分配等 ⑥
│   ├── オープン型証券投資信託の収益の分配等 ⑦
│   ├── 外国株式等の配当等 ⑧  ← ⑨合計・⑰差引金額と混同しないこと
│   └── 合計 ⑨
└── 上記以外のもの（⑩〜⑮）
    ├── 公社債の利子等 ⑩
    ├── 社債的受益権の収益の分配等 ⑪
    ├── 投資信託等の収益の分配等 ⑫
    ├── オープン型証券投資信託の収益の分配等 ⑬
    ├── 国外公社債等の配当等 ⑭  ← ⑧外国株式等と別フィールド
    └── 合計 ⑮
```

### 2. プロンプトにフィールド対応表を追加

丸数字（④〜⑲）と JSON フィールドの対応を明示:

- ⑯ 譲渡損失の金額（控除額）
- ⑰ 差引金額（⑨＋⑮－⑯）← ⑧や⑭の個別値ではない
- ⑱ 納付税額（所得税・地方税）
- ⑲ 還付税額

### 3. その他追加

- 合計フィールドに `還付税額` を追加（⑲）
