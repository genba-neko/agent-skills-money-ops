# e-Tax xtx → JSON 化基盤 + 仕様書調査

## 対象 issue

[#59](https://github.com/genba-neko/agent-skills-money-ops/issues/59)

---

## 目的

1. **各年の確定申告情報を記録として残す** — 過去申告の永続保管・参照
2. **確定申告の自動入力** — 来年度の `tax-etax` (e-Tax 作成コーナー入力) での値転記、`tax-calc` / `tax-review` での前年値参照・差分比較

## 背景

### 復元可能性調査結果 (実測)

| ファイル | 形式 | 復元可能性 |
|---|---|---|
| `r6syotoku_kessan_*.data` | NTA 作成コーナー独自形式 (マジック `eMHO`) | × 暗号化、仕様非公開 |
| `r6syotoku_*.pdf` | PDF | △ テキスト抽出可能だが座標解析必要 |
| `JyusinData*.xtx` | **平文 UTF-8 XML、84KB** | **◎ そのまま JSON 化可能** |

→ **xtx を一次データソースとして採用**

### サンプル xtx 構造 (R6 所得税)

- namespace: `http://xml.e-tax.nta.go.jp/XSD/shotoku` (所得税)
- フォーム版: VR=24.0.0
- 要素数: 2527
- 帳票 (form id):
  - `KOA020-1` 第一表
  - `KOA050-1` 第二表
  - `KOA210-1` 財産債務調書
  - `KOB060-1`, `KOB060-2` 青色申告決算書
  - `KOB240-1`, `KOB550-1`, `KOB560-1`, `KOB565-1`, `KOB975-1` 青色関連明細
  - `KOC080-1` 分離課税 第三表 (株式譲渡)
  - `KOE040-1` 明細 (寄附金 or 医療費)
  - `TEA060-1` 添付書類
  - `TEG700.../TEG830...` 添付書類 (受信通知等)
  - `IT` ヘッダ (税務署/受付情報)
- フィールドコード体系: BFC / DHC / AFL / WML / BXI / EHQ / AMF / 他多数
- XMLDSig 署名付き (`dsig:Transform`)

ユーザーの確定申告には以下が含まれる:
- 青色申告 (損益・貸借・月別売上)
- 医療費控除 (明細書)
- 株式関連 (分離課税・譲渡所得)
- 寄附金控除 (ふるさと納税)

## 仕様書調査結果 (実測、令和7年10月30日付配布)

### 配布元
- 公式: https://www.e-tax.nta.go.jp/shiyo/shiyo3.htm
- 項番9: 「XML構造設計書及び帳票フィールド仕様書【所得税関係】」
- ファイル: `e-tax09.CAB` (約8.1MB、CAB 形式)
- ライセンス: 商用・非商用利用可、財務会計ソフト開発に使用可。著作権あり (仕様書原本のリポジトリコミットは不可)

### 中身
全58ファイル、すべて Excel (.xlsx):
- 「XML構造設計書(所得-申告)Ver1x〜Ver23x.xlsx」 (23ファイル) — 階層構造 + フィールドコード
- 「XML構造設計書(所得-申請)Ver1〜21x.xlsx」 (6ファイル) — 申請系 (申告とは別系統)
- 「帳票フィールド仕様書(所得-申告)Ver1x〜Ver23x.xlsx」 (23ファイル) — 桁数・必須・補助情報
- 「帳票フィールド仕様書(所得-申請)Ver1x〜Ver21x.xlsx」 (6ファイル)

### XML 構造設計書 列構造 (Ver23x で実測)
| 列 | 意味 |
|---|---|
| c0 | 項番 |
| c1 | レベル (階層深さ) |
| c4-c11 | 階層別項目名 (人間可読: 「氏名」「住所」「年分」等) |
| c12 | 入力型 (`kubun` / `zipcode` / `address` / `name` 等) |
| c16 | ENUM 名 (`NENBUN` / `NOZEISHA_ADR` 等) |
| c17 | page (帳票識別: `KOA020-1` 等) |
| **c18** | **フィールドコード (`ABA00010` 等) = xtx タグ名と一致** |

→ **コード→項目名の自動マッピング JSON が生成可能**。手作業不要。

### Ver と xtx の VR の対応
- サンプル xtx は VR=24.0.0
- 仕様書最新は Ver23x まで → 完全一致しないが、近似で検証必要
- Ver24 は未公開の可能性 (令和7年10月時点)
- Ver23 で大半カバー、差分は実測で吸収

### 工数再評価
仕様書あり前提で **楽勝側に確定**:
- mapping JSON 自動生成: 1日
- parser 実装: 半日
- 実機検証: 半日
- 合計 約2日 (全帳票網羅)

## 注意事項

- xtx は **氏名・住所・マイナンバー・所得金額** を含む可能性大 → ログ出力時の慎重さ要 (期間中の標準出力でも個人情報を出さない)
- `data/` 既存ルール (コミット禁止) で xtx 自動カバー
- `.work/` 配下のパスはプラン・コード・コメントに記載禁止
- 1 サンプル (R6) のみで設計するが、年度 (R5/R7) 違いがあれば構造差を後追いで吸収
- XMLDSig 署名検証は本 issue では実施しない (改ざん検出用、自家保管では不要)
