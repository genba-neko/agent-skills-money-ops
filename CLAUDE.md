# CLAUDE.md — agent-skills-money-ops

## プロジェクト概要

Claude Code と Chrome 拡張連携を使い、確定申告・クレカ明細・ポイント管理などの個人財務業務を自動化するAgent Skillsプロジェクト。

---

## プロジェクト構成

```
agent-skills-money-ops/
├── plugin.json
├── CLAUDE.md
├── .gitignore
├── plan/              # プラン保存（git管理）
├── skills/
│   ├── tax-collect/   # 証券・FX・CF等から年間報告書収集
│   ├── tax-calc/      # 所得・税額計算
│   ├── tax-etax/      # e-Tax記入・提出
│   └── tax-review/    # 申告前最終確認
├── data/              # ★コミット禁止（個人情報）
├── output/            # ★コミット禁止
├── src/
│   └── money_ops/     # 共通Pythonパッケージ
└── tests/
    └── fixtures/      # モックデータ（個人情報を含まないサンプルのみ）
```

---

## 作業分担

このプロジェクトは **人間 / Claude** の2者で役割分担する。

| 役割 | 担当 |
|---|---|
| プラン初期案・設計 | Claude |
| 設計説明・レビュー | Claude |
| ブランチ名提案 | Claude |
| ブランチ名決定 | 人間 |
| 実装（コード生成） | Claude |
| テスト | Claude |
| コミット・マージ判断 | 人間 |

---

## プラン作成

- プランは `plan/` に保存する（git 管理・GitHub 上で参照可能）
- ファイル名: `YYYYMMDD_HHMM_概要.md`（**必ずコマンドで現在時刻を取得。推測・概算禁止**）
  ```bash
  date +%Y%m%d_%H%M
  ```
- プラン作成指示を受けたら **【プラン作成モード開始】** を応答してから作業開始

### フロー

1. **初期プラン案作成**: Claude が案を作成
2. **プラン調整**: 人間と対話しながら更新
3. **プラン完成**: 人間がプラン完成を宣言したら完成

---

## Issue 登録

- プランの内容を独立処理できる単位に分割して GitHub issue に登録する
- issue 本文に「関連資料」セクションを設け `plan/` ファイルへのリンクを記載する
- 未実装: Open / 実装済み: Closed
- ラベル: `feat` / `fix` / `refactor` / `test` / `chore` / `docs`

---

## 実装フロー

1. issue 番号を決めてからブランチを切る
2. 設計・実装・テスト: Claude
3. **プラン完了マーク（マージ前必須）**: `plan/` の該当 issue に `[完了 PR#XX YYYY-MM-DD]` を追記してコミット
4. push → PR → master へマージ → ローカル `git pull`

- **プランと実態が乖離した場合は `plan/` ファイルを即時更新する（人間への確認不要）**
- **マージ後の後追いプラン更新は禁止**

---

## ブランチ・コミット規則

- ブランチ命名: `feature/[issue番号]_[概要]` / `fix/[issue番号]_[概要]`
- コミットメッセージ: Conventional Commits形式（`feat:` / `fix:` / `chore:` 等）
- `git add .` および `git add -A` は**禁止**。ファイルは個別指定すること

---

## コミット禁止ファイル

以下はいかなる状況でもコミット禁止:

- `data/` — 個人の取引データ・明細
- `output/` — 生成物
- `*.config.yaml` / `*.config.json` — 認証情報・個人設定
- `.env` / `.env.*`
- ログインID・パスワードを含む一切のファイル

---

## 破壊的操作の鉄則

**削除・上書き・切り捨てを伴う設計・実装をするときは、コードを書く前に人間へ危険性を説明し、安全側に倒した設計を提案すること。**

1. **事前説明**: 削除・上書き・データ消失リスクに気づいたらコードを書く前に人間に伝える
2. **安全側に倒す**: 迷ったら削除しない・上書きしないほうを選ぶ
   - 削除の前に移動・アーカイブ
   - 上書きの前にバックアップ
3. **2ステップ操作の順序保証**: 「保存してから削除」など、保存が成功しない限り削除を実行しない設計にする

---

## セキュリティ・個人情報ルール

- ログインID・パスワードはスクリプト内に**ハードコード禁止**。環境変数または別管理の設定ファイルから読み込む
- マイナンバー・口座番号・証券口座番号はいかなるファイルにも**記載禁止**
- ログ出力時、個人情報（氏名・住所・マイナンバー・口座番号）を**出力禁止**

---

## ブラウザ自動操作ルール

- 各操作間は **1〜3秒のwait** を入れること（レート制限・BAN対策）
- Playwright使用時は `headless=False` で動作確認してから headless 化する
- **2FA・CAPTCHA が出たら即座に処理を停止し、ユーザーに手動介入を依頼する**
- 並列収集（Subagents）に対応できる設計にすること

---

## Python開発ルール

- パッケージ構成: `src/money_ops/` + `tests/`
- 依存パッケージは `pyproject.toml` で管理する
- テストは `pytest` を使用する
- テスト用モックデータは `tests/fixtures/` に置く（個人情報を含まないサンプルのみ）

---

## スキル開発ルール

- 各スキルは `skills/<skill-name>/SKILL.md` に仕様・使い方を記述する
- スキル名は prefix でドメインを示す（`tax-*`, `expense-*` 等）
- 新スキル追加時は `plugin.json` の `skills` リストにも追加する
